# Telegram-бот для скачивания медиа-контента

Бот для скачивания видео и фото по ссылкам с поддержкой TikTok, YouTube, Instagram, Pinterest. Включает учёт пользователей, лимиты, админ-панель и рассылки.

## Требования

- Python 3.10+
- FFmpeg (для объединения аудио/видео при скачивании с YouTube и др.)
- Токен бота от [@BotFather](https://t.me/BotFather)

## Установка

1. Клонируйте репозиторий или скопируйте файлы проекта.

2. Создайте виртуальное окружение и установите зависимости:

```bash
python -m venv venv
venv\Scripts\activate   # Windows
# или: source venv/bin/activate  # Linux/macOS
pip install -r requirements.txt
```

3. Установите [FFmpeg](https://ffmpeg.org/download.html) и добавьте его в PATH.

4. Скопируйте `.env.example` в `.env` и заполните:

```env
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_IDS=123456789,987654321
DAILY_DOWNLOAD_LIMIT=3
FLOOD_DELAY_SECONDS=1.5
```

- **ADMIN_IDS** — Telegram ID администраторов через запятую (узнать: [@userinfobot](https://t.me/userinfobot)).

## Запуск

```bash
python main.py
```

## Функционал

### Пользователи

- **Команда /start** — приветствие, кнопки «Подписаться» и «Проверить подписку».
- Проверка подписки на данном этапе не выполняется: после нажатия «Проверить подписку» доступ к боту открывается.
- Каждый новый пользователь записывается в БД (Telegram ID, username, дата первого запуска).
- Пользователь отправляет ссылку → бот определяет платформу, скачивает контент и отправляет файл.
- Лимит: 3 загрузки в сутки (настраивается в `.env`).
- Ограничение флуда: задержка 1–2 сек между запросами одного пользователя.

### Поддерживаемые платформы

- **Видео:** TikTok, YouTube (в т.ч. Shorts), Instagram (Reels, посты), Pinterest.
- **Фото:** Instagram, Pinterest.
- **Stories:** Instagram Stories (публичные аккаунты).

Используется [yt-dlp](https://github.com/yt-dlp/yt-dlp).

### Админ-панель

- Команда **/admin** (только для ID из `ADMIN_IDS`).
- **Статистика пользователей:**
  - за день (конкретная дата 00:00–23:59);
  - за месяц (текущий календарный месяц);
  - за всё время.
  - Кнопка «Обновить статистику».
- **Рассылка:** текст, фото, видео или фото/видео + текст; неограниченное количество inline-кнопок (текст и URL). Кнопки в формате: `Текст кнопки | https://ссылка.com`. Завершение вводом слова «готово».
- **Каналы подписки:** добавление каналов (@username или ID), включение/выключение проверки подписки (на данном этапе проверка не обязательна).

## Структура проекта

```
├── main.py           # Точка входа
├── config.py         # Конфигурация и тексты
├── keyboards.py      # Inline-клавиатуры
├── requirements.txt
├── .env.example
├── data/             # Создаётся при первом запуске (БД, загрузки)
├── database/        # Работа с SQLite (пользователи, загрузки, каналы)
├── handlers/         # Обработчики: start, download, admin
└── services/        # Скачивание (yt-dlp), ограничение флуда
```

## База данных

SQLite (`data/bot.db`). Таблицы: пользователи, загрузки (для лимита по дням), каналы подписки, настройки. При первом запуске таблицы создаются автоматически.

## Реферальная система

В ТЗ указано «реферальная система остаётся без изменений» — в коде зарезервирована таблица `referral_codes`; логику рефералов при необходимости можно доработать отдельно.
